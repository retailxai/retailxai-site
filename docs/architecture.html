Architecture Overview

RetailXAI is built with a modular, pipeline-first architecture designed for extensibility and production reliability.

High-Level Design

The system follows a pipeline-first architecture with clear separation of concerns:

  • Pipeline Orchestrator: Single entry point for content processing
  • Service Layer: Business logic for content processing
  • Agent System: Advanced workflows (feature-flagged)
  • Infrastructure: Database, logging, circuit breakers, cost tracking

System Flow

CLI → Pipeline Orchestrator → Services → Database/APIs

Each stage processes content and passes results to the next stage.

Core Components

1. Pipeline Orchestrator

The Pipeline Orchestrator is the single source of truth for content processing. It coordinates:

  • Video/content ingestion
  • Transcription
  • Content analysis
  • Article drafting
  • Publishing

2. Service Layer

Business logic is encapsulated in dedicated services:

  • Ingest Service: Video metadata and transcript extraction
  • Auto Tagger: Content tagging with AI
  • Video Classifier: Content type classification
  • Summary Extractor: Key points extraction
  • Article Draft Service: Article generation

3. Fallback Strategy

Critical services implement fallback chains for reliability:

  • Transcription: Local Whisper → OpenAI Whisper → Google Speech
  • AI Services: Primary model → Secondary model → Graceful degradation

Data Flow

  1. User submits content URL via CLI
  2. Pipeline ingests content and extracts metadata
  3. Transcription service converts audio to text
  4. Analysis services extract tags, summaries, and classifications
  5. Article generation creates professional content
  6. Publishing service distributes to platforms

Key Design Principles

1. Async-First Architecture

All I/O operations are asynchronous for better resource utilization and scalability.

2. Structured Logging

All components use structured JSON logging for easy analysis and debugging.

3. Circuit Breaker Pattern

API calls are protected by circuit breakers to prevent cascade failures.

4. Cost Tracking

All AI API calls are tracked for cost management and optimization.

Security

  • API keys stored in environment variables
  • No persistent storage of user data
  • Input validation and sanitization
  • GDPR-compliant data handling

Extensibility

The architecture supports easy extension:

  • Add new services by implementing standard interfaces
  • Extend agent tools through the tool registry
  • Add publishing platforms via adapter pattern
